# 条目系统

## 最小规范

一句话：

```text
很多条目, 能搜
```

稍微展开一点：

```text
有很多条目。
可以查询/搜索它们。
结果仍然是条目。
```

最小规范不定义文件、日期、标题、标签、链接、反链、TODO、Markdown、
Org、数据库、排序、高亮、同步、发布或 UI。

这些都是具体实现的选择。写成代码式表达就是：

```text
result = query(store)
```

`store` 是很多条目。`result` 是查询返回的条目。

## 文件实现

本地文本文件实现用 Markdown 文件存储条目，并按日期分片：

```text
root/YYYY/MM/YYYY-MM-DD.md
```

例如：

```text
~/Drafts/memo/2026/05/2026-05-29.md
```

文件只是存储分片，不是语义对象。日期路径只回答一个问题：这条内容物理上存在哪里。

## 条目格式

因为这个实现使用 Markdown，每个条目用一个时间戳标题开始：

```md
# 2026-05-29 14:20

content, [[search token]], content
```

这个标题只是条目边界，不是核心模型里的“标题”概念。

从这个标题之后到下一个时间戳标题之前，都是这个条目的内容。条目可以是一句话，也可以是一篇长文。

## 搜索标记

`[[...]]` 是文件实现里的文本约定，用来标出一段值得搜索的字符串：

```md
[[纯文本系统]]
```

它不是核心模型里的链接。在 vis 实现里，光标放在这个标记上按 `<Enter>`，
会对里面的文本做固定字符串全文搜索。

如果同一段文本出现很多次，这不是系统错误，只是这次查询返回了很多条目。

## vis 实现

vis 实现在：

```text
my/entry.lua
```

当前行为：

```text
 d              搜索当前代码项目
 g              搜索历史访问过的所有文件
 f              列出当前代码项目文件
 j              全局追加新笔记条目
 l              全局列出/搜索所有笔记条目
<Enter>         搜索当前上下文；在 [[...]] 上优先搜标记内容
gf              打开光标下文件/目标
gx              打开光标下 URL
 h              历史文件
 b              返回上一个文件
:entry-new      追加新条目，命令参数会作为内容
:entry-search   用 rg + fzf 搜索条目
```

写代码时，当前路径就是代码项目路径：

```text
d      rg 搜项目
g      rg 搜历史访问过的所有文件
f      fzf 列文件
Enter  搜当前词/选区
```

笔记功能是全局的，不重载 `f/d`：

```text
j      追加新条目到今天文件末尾
l      fzf 列出所有条目，可直接搜条目全文
Enter  在 [[...]] 上搜标记内容；否则搜当前词/选区
```

` l` 只列出 `# YYYY-MM-DD HH:MM` 条目边界。列表按时间倒序排列，
界面显示时间和摘要，但条目全文也会作为隐藏字段交给 fzf，所以可以直接搜条目内容。

` j` 全局直接追加新条目到今天文件末尾，并把光标放到正文位置。

`<Enter>` 是全局上下文搜索入口：在条目仓库里，光标位于 `[[...]]` 上时搜索
里面的文本；没有 `[[...]]` 或不在条目仓库时，退回到普通项目的当前词/选区搜索。

全文搜索使用 `rg --fixed-strings`，结果选择使用 `fzf`。选择结果后会打开对应文件和行。

这个实现刻意不维护索引，不解析反链，不要求标题唯一，也不解释标签。
